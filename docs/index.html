Attention

Understanding Each State

🔇 Ambient State: No Machine Activity

The machine is powered off or no meaningful mechanical activity is detected. The system remains active and listening, but no significant vibration or acoustic signature is present.

Signal Characteristics: The PeakHz values are extremely volatile—swinging wildly across the full 0–2000 Hz range with no stable pattern. This chaotic behavior is characteristic of background environmental noise with no organized mechanical source. RMS and Centroid values also shift unpredictably.

What to do: Monitor the device. Ambient state is normal for powered-off or idle equipment. No action needed.

✅ Healthy State: Normal Operation

The machine is running normally. Signal features are stable and within expected bounds. No intervention is needed at this time.

Signal Characteristics: The PeakHz line sits in a tight, stable band (typically 650–700 Hz) for most of the monitoring period. RMS and Centroid move gradually and smoothly without sudden jumps. Natural variation appears as load shifts—but always stays within the expected envelope. The overall picture is calm, consistent, and predictable.

What to do: Continue normal operations. Maintenance is scheduled as per standard intervals. Equipment is operating as expected.

⚠️ Fault State: Anomaly Detected

An anomaly has been detected. The signal shows characteristics that deviate from the healthy baseline. The system flags this early so action can be planned rather than forced.

Signal Characteristics: The PeakHz line is mostly stable but interrupted by sharp, isolated spikes that shoot to 1700–2000 Hz before returning to baseline. These sudden impulse events are the key fault indicator. Unlike Ambient noise (which is continuously chaotic), these spikes are brief, repeating, and structurally distinct. RMS and Centroid may dip slightly around fault events, indicating abnormal stress.

What to do: Plan a maintenance inspection soon. The equipment is showing early signs of wear or developing mechanical issues. Address before conditions worsen.

🖥️ Display States Explained

Physical Platform

Hardware components

Carefully selected components that balance capability, cost, and reliability in field environments.

⚙️ ESP32 Controller

Powerful microcontroller runs all on-device monitoring, analysis, and display workflow. Low power consumption perfect for continuous operation.

🎤 Digital MEMS Mic

Precision microphone captures machine sound signatures and vibration harmonics for continuous real-time observation and pattern analysis.

📺 SSD1306 OLED

128×64 monochrome display presents live machine condition directly at the edge. Always available, no network dependency.

🛠️ Implementation Philosophy

All computation and decision-making happens on the edge device itself. This means:

  • ✅ No cloud dependency or latency
  • ✅ Real-time response always available
  • ✅ Works in offline environments
  • ✅ Data never leaves the facility
  • ✅ Low bandwidth requirements
